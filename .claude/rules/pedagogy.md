# Pedagogy Rules

## Design Principles
- Build understanding incrementally — introduce one concept per visual beat
- Transitions should explain, not merely decorate
- Every animation should answer "what should the student notice right now?"
- Prefer concrete examples before abstract formulas

## Pacing
- Allow 0.3-0.5s wait after each new element appears
- Allow 0.7-1.0s wait after key revelations or equation transformations
- Keep total scene duration under 3 minutes for focused topics
- Group related animations with LaggedStart for visual cohesion

## Visual Clarity
- Maximum 5-7 distinct elements on screen simultaneously
- Use color consistently within a scene (same object = same color)
- Ensure text is readable at 480p (minimum font_size=20 for annotations)
- Use SurroundingRectangle or Indicate to draw attention to key elements

## Mathematical Notation
- Use standard mathematical conventions for notation
- Break complex formulas into TransformMatchingTex steps
- Show intermediate computation steps, don't skip to the answer
- Label all variables and expressions on first appearance
